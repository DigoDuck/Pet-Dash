- [ ] **Step 1: Adicionar o bloco de produção ao final do settings.py**

Acrescentar ao final de `backend/config/settings.py`:

```python
CSRF_TRUSTED_ORIGINS = env.list("CSRF_TRUSTED_ORIGINS", default=[])

if not DEBUG:
    SECURE_SSL_REDIRECT = env.bool("SECURE_SSL_REDIRECT", default=True)
    SECURE_PROXY_SSL_HEADER = ("HTTP_X_FORWARDED_PROTO", "https")
    # O healthcheck da Railway chega sem X-Forwarded-Proto; sem a isenção ele
    # levaria 301 e o deploy nunca ficaria healthy.
    SECURE_REDIRECT_EXEMPT = [r"^api/health/$"]
    SECURE_HSTS_SECONDS = 2_592_000  # 30 dias; subir para 1 ano com o deploy estável
    SECURE_HSTS_INCLUDE_SUBDOMAINS = True
    SESSION_COOKIE_SECURE = True
    CSRF_COOKIE_SECURE = True
```

Racional (para o relatório de estudos): `SECURE_PROXY_SSL_HEADER` diz ao Django para confiar no header do proxy da Railway ao decidir se a request é HTTPS; sem isso o `SECURE_SSL_REDIRECT` entra em loop de redirect. `SECURE_SSL_REDIRECT` é env-driven para o smoke local (Task 6) poder desligá-lo.
